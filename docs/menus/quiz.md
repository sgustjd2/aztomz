# 유행어 퀴즈

## 목적
신조어 뜻 맞히기로 내 MZ력 테스트. 출처로 검증한 정확한 뜻만 사용해 바이럴 테스트와 차별화.

## 진입·화면
- **파일:** `frontend/quiz.html`
- **진입:** 마스트헤드 "유행어 퀴즈" 링크 또는 dictionary.html의 CTA ("🎮 내 MZ력은 몇 점?")
- **형식:** 4지선다 × (최대 8문제) — 뜻(힌트) 표시 → 단어 선택 → 정답 해설 + 등급 계산

## 표시 데이터
- `t.def` (필수) — 한 줄 뜻(단어 미포함). 힌트로 사용
- `t.title` — 정답 단어
- `t.pureKorean` (선택) — 예쁜 우리말. 정답 해설에 표시
- `t.stage` (선택) — 유행 단계. 문제 카드에 보조 정보로 표시
- `t.id` — 정답 링크(해설에서 클릭 시 trend.html로 이동)
- 데이터 원본: `trends.json`의 cat-slang 또는 type=='신조어' 항목들 (def 필드 필수)

## 현재 상태
구현됨. `quiz.html`에 풀이 엔진 완성. 신선도 게이트 적용: 한물간 신조어(fresh:false 또는 analyzedAt 14일+ 미갱신)는 자동 제외. 따라서 풀에는 항상 최신 신조어만 포함됨. 현재 풀 크기 ~22개(신선도 통과한 단어들, 모두 def 필드 포함). 주간 갱신(Hermes hangeut-run 스킬)으로 유지.

**점수·등급:**
- MZ력 100% — "🔥 찐 영크크" (정답 8/8)
- MZ력 75~99% — "✨ 영크크 인정"
- MZ력 50~74% — "😅 애매크크"
- MZ력 25~49% — "👀 늙크크 주의보"
- MZ력 0~24% — "👴 늙크크 확정"

## 관련 코드
- `frontend/quiz.html` — 화면 마크업 + 풀이 엔진 (L81~189)
  - `build()` — 문제 세트 생성 (최대 8개, def 있는 항목만)
  - `question()` — 진행률·현재 점수·선지 렌더링
  - `pick()` — 선택지 처리 + 정답/오답 표시
  - `result()` — 최종 점수·등급·공유 버튼·정답 해설
- `frontend/assets/app.js`
  - `H.share()` (L156~166) — 점수 자랑 공유 (네이티브 공유 시트 → 링크 복사 폴백)
  - 데이터 필터: `H.TRENDS.filter(t=>t.coverCat==='cat-slang' || t.cat==='신조어')`
- `backend/data/trends.json` — cat-slang 항목들의 def 필드

## 비고
- **def 필드:** 필수. 없으면 해당 단어는 풀에서 제외
- **신선도 게이트:** `H.isFreshSlang(t)` 조건
  - `t.fresh === false` → 제외 (사람이 명시적으로 '한물'으로 마킹)
  - `t.stage` 포함 '끝물'|'한물'|'지남'|'옛유행'|'outdated' → 제외
  - `t.analyzedAt` 또는 `t.collectedAt`이 14일+ 이전 → 제외
  - **목적:** 항상 최신 신조어만 출제. 2026년 상반기 신조어 유행 속도에 맞춰 오래된 신조어 자동 회전
- **공유 기능:** 결과 공유 → `H.share()`로 구현. navigator.share 미지원 환경에선 링크 복사 + 토스트 메시지
- **2차 기능 후보:**
  - 친구와 점수 대결 (URL 인자로 상대방 점수 전달)
  - 카테고리별 퀴즈 (예: "성수 맛집 특화")
  - 난이도 선택 (상·중·하)
