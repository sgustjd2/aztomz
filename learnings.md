## 소스·도구
- [2026-06-05] (소스) 인스타/틱톡/페북 **직접 접근 불가**(ToS·차단). ddgs 검색 공개글 + 유튜브 API(조회수·댓글)만.
- [2026-06-05] (소스) Threads에서 "요즘 유행하는 ○○" 형태 글이 최신 트렌드 잘 잡힘.
- [2026-06-05] (소스) 브라우저(Chrome) 안 됨. ddgs.exe / youtube_search.py 만.
- [2026-06-05] (소스) **출처 URL은 검색결과에 실제로 나온 것만** 쓸 것. 긴 URL을 짧게 줄이거나 추측해서 만들면 404 난다(실제로 보그·W코리아 가짜URL이 404였음). `node scripts/check-links.mjs`로 점검.
- [2026-06-05] (소스) **200이어도 본문이 그 트렌드를 안 다루면 가짜 출처.** `ddgs extract -u URL -f text`로 본문에 핵심어가 실제 나오는지 확인(hangeut-source 에이전트). 살아만 있고 무관한 링크 = 신뢰 깎임.
- [2026-06-05] (소스) **본문 관련성 도구: `node scripts/check-source.mjs [파일] [--only=id,id]`** — 각 src 본문을 받아 트렌드 키워드 수로 정확✅/약함⚠️/무관❌/차단🚫 자동 판정(LLM 안 씀). check-links(상태)의 짝.
- [2026-06-05] (소스/함정) **ddgs를 그냥 부르면 Windows cp949로 출력하다 본문에 한자·키릴·악센트가 나오면 크래시→본문이 잘려** 한글 키워드가 0으로 보인다(보그·문화일보가 거짓 '무관'으로 찍혔던 원인). `PYTHONIOENCODING=utf-8`/`PYTHONUTF8=1` 강제 필수. check-source.mjs는 자동 처리.
- [2026-06-05] (소스) **다이닝코드 profile.php·인스타그램 페이지는 JS앱/로그인벽이라 ddgs가 본문을 못 읽어 '무관/차단'으로 뜬다**(들쭉날쭉). 출처는 tistory·trip.com·티스토리계열·언론기사처럼 **본문이 실제로 읽히는 링크**로 쓸 것. (누데이크·멜로워·이스트베이글 출처를 블로그/기사로 교체함.)

## 운영·파이프라인
- [2026-06-11] (운영) Hermes 기본 모델(gemini-2.5-flash, 구글 무료 OAuth)은 21시대에 구글측 용량 429가 잦음 — 6/10은 재시도로 살았고 6/11은 6회 재시도 전부 실패해 크론이 죽음. `fallback_providers` 폴백을 시험했으나 **둘 다 실효성 없음 확인**: OpenRouter 보유 키 3개는 크레딧 $0 무료 계정(유료 모델 402/403, 1번 키는 일일 한도 초과), Anthropic OAuth는 서드파티 앱이 extra usage 크레딧 필요(400). → **Gemini 단독 운용으로 확정**(`fallback_providers: []`). 429로 죽은 날은 `hermes cron run 9ddacd750b48`로 수동 재실행. (참고: 크론도 폴백을 탐 — 공식 문서의 "크론 미지원"은 낡은 내용, scheduler.py가 config를 실행 시점마다 읽음.)
