## ① 두바이 초콜릿 (편의점판) — *근거 충분*

**5분 리서치:** CU 출시 하루 20만개 완판 / 숏폼 ASMR 먹방·협찬 영상 다수 /
원조(Fxx Dessert Chocolatier "Can't Get Knafeh of It") 대비 혹평 — 홍사운드 등 "그냥 화이트초콜릿",
팜유 준초콜릿·색소·카다이프 대신 튀긴 소면 지적.

**광고 의심도 체크**
- ☑ 비슷한 문구/구도 반복(자르는 소리·비주얼 ASMR 포맷)
- ☑ 협찬·광고 콘텐츠 존재(유튜브 "광고" 표기)
- ☑ 출시 직후 단기 집중 노출
- ☑ 인플루언서(179만 유튜버 등) 중심 확산
- ☐ 구체 경험 설명 부족 → 오히려 풍부, 체크 안 함
- ☐ 지도-SNS 괴리 → 편의점 제품이라 지도 개념 약함, **판단 보류**
- ☐ 부정 후기 과소 → 부정 후기 많음, 체크 안 함
→ **체크 4/7 → 광고 의심도 57점 (중간)**

**실제 후기 신뢰도 체크**
- ☑ 구체적 맛·식감 표현(와사삭, "화이트초콜릿 맛", 팜유)
- ☑ 가격 만족/불만 다양(1개 4000원 비싸다 등)
- ☐ 웨이팅 → n/a, **보류**
- ☐ 재방문 의사 → 확인 안 됨, **보류**
- ☑ 단점 언급 풍부(명확한 혹평)
- ☑ 일반 사용자 리뷰 비중(스레드 등 일반 계정)
- ☐ 리뷰 기간 분산 → 출시 시점 집중, 체크 안 함
→ **체크 4/7 → 실제 후기 신뢰도 57점 (보통)**

**최종 라벨:** 중간 광고 + 보통 신뢰 → "취향 따라 갈림" 인데… → ⚠️ **모델 결함 발견(아래 C 참고)**

**한끗 판단:** 바이럴·협찬은 강했지만, *풍부하고 신뢰할 만한* 실제 후기들이 "편의점판은 원조와
다르고 가성비 별로"라고 **일관되게 부정적**으로 말합니다. 즉 후기 데이터는 믿을 만한데 내용이 안 좋아요.
호기심에 한 번은 OK, 원조 기대는 금물.

## ② 누데이크(NUDAKE) 성수 — *중간 근거 (장소라 신호 일부 보류)*

**5분 리서치:** 젠틀몬스터 산하 디저트, "MZ 제일 핫"·비주얼 화제 / 1인 7,000~15,000원 /
점심 주문 웨이팅 있으나 테이블 웨이팅 짧음 / "기괴+힙한 공간", "가격대·맛 한번이면 족" 평.

**광고 의심도 체크**
- ☑ 비주얼/포토 목적 칭찬 반복("비주얼 사로잡는")
- ☐ 협찬 해시태그 → 확인 안 됨(브랜드 자체 마케팅 강함), **보류**
- ☐ 단기 집중 → 장기 운영, 체크 안 함
- ☑ 화제성·인플루언서 중심
- ☑ 맛보다 분위기·비주얼 칭찬 위주
- ☐ 지도-SNS 괴리 → 다이닝코드 평점 존재, **보류**
- ☐ 부정 후기 과소 → "한번이면 족" 솔직 평 있음, 체크 안 함
→ **체크 3/7 → 광고 의심도 45점 (중간)** *(주: 협찬 거품이라기보단 브랜드 비주얼 마케팅형)*

**실제 후기 신뢰도 체크**
- ☑ 구체적 맛 표현(마이크로 크로아상 "한올한올", 피에타 라떼 비주얼)
- ☑ 가격 언급(7~15k, "한번이면 족")
- ☑ 웨이팅 경험(점심 주문 웨이팅/테이블 짧음)
- ☐ 재방문 의사 → "한번이면 족"=재방문 약함, 체크 안 함
- ☑ 단점 언급("기괴", "한번이면 족")
- ☐ 일반 계정 비중 → **보류**
- ☐ 리뷰 기간 분산 → **보류**
→ **체크 4/7 → 실제 후기 신뢰도 57점 (보통)**

**최종 라벨:** 중간 광고(45) + 보통 신뢰(57) → "SNS 기대치 조절 / 취향 따라 갈림"

**한끗 판단:** 비주얼·공간 경험형이에요. 맛·가성비보다 "힙한 공간 한 번 가보는" 목적에 맞습니다.
- 사진/힙스팟: 추천 · 맛·재방문 기대: 낮추기 · 부모님 동반: 보통 · 웨이팅 싫음: 평일 오전

## C. ⭐ 손분석에서 발견한 PRD 모델 결함 (중요)

**문제:** PRD 9.7 라벨표는 "실제 후기 신뢰도 높음 = 좋은 곳"으로 가정한다. 그런데
**"후기가 믿을 만한가(신뢰도)"와 "실제로 좋은가(만족도)"는 다른 축**이다.
두바이 초콜릿 편의점판이 정확히 이 함정: 후기는 풍부·구체적·신뢰할 만한데 **내용은 부정적**.
이걸 "신뢰도 높음 → 믿고 가볼 만함"으로 라벨하면 정반대로 오도한다.

**수정 제안(v0 룰셋에 반영):**
- 축을 셋으로: **① 광고 의심도** + **② 후기 신뢰도(믿을 만한가)** + **③ 만족도/방문가치(좋다는가 나쁘다는가)**.
- 또는 최소한 "한끗 판단" 문장에서 *신뢰도와 평가 방향을 분리*해 적는다(두바이 케이스처럼).
- → [`template.md`](template.md)에 "③ 만족도(긍정/부정/혼재)" 한 줄 추가 권장.

이게 바로 **코드 짓기 전 손분석으로 잡아야 했던 결함**이다. hermes 자동화에 이대로 넣었으면
"신뢰도 높은 별로인 집"을 "믿고 가세요"로 추천하는 사고가 났을 것.
